## 概览

Omnara API 使用基于 JWT 的 Bearer 令牌认证。所有 API 请求都必须在 Authorization 请求头中包含有效的 API key。

## 获取 API key

### 通过 CLI

```bash
omnara --auth
```

你的 API key 会自动保存到 `~/.omnara/credentials.json`。

### 通过仪表板

1. 访问 [claude.omnara.com/dashboard](https://claude.omnara.com/dashboard)
2. 进入 **Settings** → **API Keys**
3. 点击 **Generate New Key**
4. 复制密钥（密钥只会显示一次）

<Warning>
请妥善保管你的 API key。它们拥有对你 Omnara 账户的完整访问权限。
</Warning>

## 使用 API key

### HTTP 请求头

在 `Authorization` 请求头中包含 API key：

```bash
curl -H "Authorization: Bearer xxxxxxxxxxxxxxxxxxxx" \
  https://agent.omnara.com/api/v1/messages/agent
```

### Python SDK

```python
from omnara import OmnaraClient

client = OmnaraClient(api_key="xxxxxxxxxxxxxxxxxxxx")
```

SDK 会自动为所有请求添加 Authorization 请求头。

### 环境变量

设置 `OMNARA_API_KEY`，避免在代码中硬编码密钥：

```bash
export OMNARA_API_KEY="xxxxxxxxxxxxxxxxxxxx"
```

```python
import os
from omnara import OmnaraClient

# 自动从 OMNARA_API_KEY 读取
client = OmnaraClient(api_key=os.getenv("OMNARA_API_KEY"))
```

## API key 格式

Omnara API key 是具有如下结构的 JWT 令牌：

- **格式：** Base64 编码的 JWT（无前缀）
- **算法：** RS256（使用 SHA-256 的 RSA）
- **声明（Claims）：**
  - `sub`：你的用户 ID
  - `iat`：签发时间戳
  - 无过期时间（永久有效）

解码后的 JWT 载荷示例：

```json
{
  "sub": "user_abc123xyz",
  "iat": 1640000000
}
```

## 密钥管理

### 列出密钥

在仪表板中查看你的所有 API key：

**Settings** → **API Keys**

每个密钥会显示：
- 名称/描述
- 创建日期
- 最近使用时间
- 状态（有效/已吊销）

### 吊销密钥

立即使密钥失效：

1. 进入 **Settings** → **API Keys**
2. 找到要吊销的密钥
3. 点击 **Revoke**

使用该密钥的所有请求会立即返回 401 Unauthorized 失败。

### 密钥轮换

安全的密钥轮换流程：

<Steps>
  <Step title="生成新密钥">
    在仪表板中创建新的 API key
  </Step>

  <Step title="更新应用程序">
    更新你的应用程序以使用新密钥：
    - 环境变量
    - 密钥管理系统
    - 配置文件
  </Step>

  <Step title="验证">
    测试应用程序在新密钥下是否正常工作
  </Step>

  <Step title="吊销旧密钥">
    验证通过后，吊销旧密钥
  </Step>
</Steps>

## 错误响应

### 401 Unauthorized

API key 缺失或无效：

```json
{
  "error": {
    "code": "unauthorized",
    "message": "Invalid or missing API key"
  }
}
```

**原因：**
- 缺少 Authorization 请求头
- 密钥格式无效
- 密钥已被吊销
- 密钥已过期（如果启用了过期机制）

**修复：**
```bash
# 验证密钥是否正确
curl -H "Authorization: Bearer YOUR_KEY" \
  https://agent.omnara.com/api/v1/auth/verify
```

### 403 Forbidden

密钥有效但权限不足：

```json
{
  "error": {
    "code": "forbidden",
    "message": "Insufficient permissions for this operation"
  }
}
```

**原因：**
- 范围化密钥缺少所需权限
- 访问了其他用户的资源

## 测试认证

验证你的 API key 是否可用：

```bash
curl -H "Authorization: Bearer YOUR_KEY" \
  https://agent.omnara.com/api/v1/auth/verify
```

成功响应：

```json
{
  "valid": true,
  "user_id": "user_abc123xyz",
  "scopes": ["read", "write"]
}
```

## 自托管实例

对于自托管的 Omnara：

```bash
export OMNARA_API_URL="https://omnara.mycompany.com"
```

```python
from omnara import OmnaraClient

client = OmnaraClient(
    api_key="your-key",
    base_url="https://omnara.mycompany.com"
)
```
