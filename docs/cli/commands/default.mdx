## 用法

```bash
omnara [OPTIONS]
```

启动一个具有完整终端集成和仪表板同步功能的 AI 编程智能体。你在终端中的所有操作都会实时流式传输到你的网页和移动端仪表板。

## 选项

<ParamField path="--agent" type="string" default="claude">
  选择要使用的智能体：`claude`、`codex` 或 `amp`
</ParamField>

<ParamField path="--name" type="string">
  在仪表板中显示的自定义名称
</ParamField>

<ParamField path="--api-key" type="string">
  用于身份验证的 API key（也可以使用 `OMNARA_API_KEY` 环境变量）
</ParamField>

<ParamField path="--no-relay" type="boolean">
  禁用 WebSocket 流式传输（仅本地会话，不进行仪表板同步）
</ParamField>

<ParamField path="--agent-instance-id" type="string">
  通过提供实例 ID 恢复已有会话
</ParamField>

<ParamField path="--base-url" type="string" default="https://agent.omnara.com">
  Omnara API 服务器 URL（用于自托管实例）
</ParamField>

## 示例

<CodeGroup>

```bash Default
# 启动 Claude Code（默认智能体）
omnara
```

```bash Codex
# 以自定义名称启动 Codex
omnara --agent codex --name "Backend Refactor"
```

```bash Local Only
# 在不进行仪表板流式传输的情况下运行
omnara --no-relay
```

```bash Resume Session
# 继续一个已有会话
omnara --agent-instance-id abc-123-def
```

</CodeGroup>

## 工作原理

1. **身份验证**：首次运行时会打开浏览器进行身份验证。API key 会保存到 `~/.omnara/credentials.json`
2. **智能体启动**：使用环境配置启动所选智能体（Claude Code、Codex 或 Amp）
3. **终端集成**：使用 PTY（伪终端）捕获所有终端输入输出
4. **WebSocket 流式传输**：将终端内容流式传输到中继服务器，实现仪表板实时同步
5. **双向通信**：来自仪表板的消息会被回传给智能体

## 支持的智能体

<Tabs>
  <Tab title="Claude Code">

    - 终端流式传输

    - 心跳跟踪
  </Tab>
  <Tab title="Amp">

    - ANSI 代码解析
    - TUI 交互支持
  </Tab>
</Tabs>

## 配置

设置默认智能体，避免每次都指定 `--agent`：

```bash
omnara --set-default codex
```

这会将你的偏好保存到 `~/.omnara/config.json`。

## 故障排查

<AccordionGroup>
  <Accordion title="身份验证失败">
    尝试重新进行身份验证：
    ```bash
    omnara --reauth
    ```
  </Accordion>
  <Accordion title="找不到智能体">
    确保已安装相应的智能体 CLI：
    - Claude Code：`claude --version`
    - Codex：随 Omnara 一起打包
    - Amp：`amp --version`
  </Accordion>
  <Accordion title="仪表板未同步">
    检查 WebSocket 连接：
    - 确认未设置 `--no-relay`
    - 检查防火墙/代理设置
    - 尝试 `--relay-url wss://relay.omnara.com/agent`
  </Accordion>
</AccordionGroup>
