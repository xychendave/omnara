## 用法

```bash
omnara headless [OPTIONS]
```

在后台运行 Claude Code，无需终端 UI。非常适合仅通过仪表板交互、自动化场景，或当你希望完全通过网页/移动应用控制 Claude 的情况。

## 选项

<ParamField path="--prompt" type="string" default="You are starting a coding session">
  会话启动时发送给 Claude 的初始提示词
</ParamField>

<ParamField path="--permission-mode" type="string">
  权限处理模式：
  - `acceptEdits` - 自动接受所有编辑操作
  - `bypassPermissions` - 完全绕过所有权限检查
  - `plan` - 规划模式（Claude 在执行前先制定计划）
  - `default` - 正常的权限提示行为
</ParamField>

<ParamField path="--allowed-tools" type="string">
  以逗号分隔的允许工具白名单（例如 `Read,Write,Bash`）
</ParamField>

<ParamField path="--disallowed-tools" type="string">
  以逗号分隔的禁用工具黑名单
</ParamField>

<ParamField path="--cwd" type="string">
  Claude 的工作目录（默认为当前目录）
</ParamField>

## 示例

<CodeGroup>

```bash Basic
# 以默认设置运行 headless 模式
omnara headless
```

```bash Auto-accept
# 自动接受所有编辑，并指定特定工具
omnara headless --permission-mode acceptEdits --allowed-tools Read,Write,Bash
```

```bash Custom Prompt
# 以特定任务启动
omnara headless \
  --prompt "Review all files in src/ for security issues" \
  --cwd /path/to/project
```

```bash Planning Mode
# 对复杂任务使用规划模式
omnara headless \
  --permission-mode plan \
  --prompt "Refactor the authentication module"
```

</CodeGroup>

## 权限模式

<Tabs>
  <Tab title="acceptEdits">
    自动接受所有文件编辑操作。Claude 仍可通过仪表板提问，但文件更改会在无需提示的情况下进行。

    **适用场景**：在可信环境中追求速度而非控制
  </Tab>
  <Tab title="bypassPermissions">
    完全绕过所有权限检查。Claude 以完全自主的方式运行。

    **适用场景**：全自动化工作流、CI/CD 流水线
  </Tab>
  <Tab title="plan">
    Claude 在执行前制定详细计划。你可以通过仪表板审阅并批准计划。

    **适用场景**：复杂重构、架构变更
  </Tab>
  <Tab title="default">
    标准权限提示。Claude 在执行需要批准的工具前会先询问。

    **适用场景**：需要人工监督的交互式会话
  </Tab>
</Tabs>

## 工具过滤

控制 Claude 可以使用哪些工具：

### 白名单（allowed-tools）
```bash
omnara headless --allowed-tools Read,Write,Bash
```
只有这些工具（外加 `mcp__omnara__approve`）可以使用，其他所有工具都会被阻止。

### 黑名单（disallowed-tools）
```bash
omnara headless --disallowed-tools Bash,Edit
```
除这些工具外，其他所有工具都可以使用。

### 组合使用
```bash
omnara headless \
  --allowed-tools Read,Write,Bash,Grep \
  --disallowed-tools Edit
```
允许工具列表优先级更高，适合进行细粒度控制。

<Tip>
`mcp__omnara__approve` 工具始终被允许，即使在严格的工具过滤下也能保证仪表板交互。
</Tip>

## 工作原理

1. **初始化**：创建一个智能体实例并连接到 Omnara MCP 服务器
2. **初始提示词**：将你指定的提示词发送给 Claude
3. **后台处理**：Claude 处理请求并将响应流式传输到仪表板
4. **消息排队**：当你通过仪表板回复时，消息会被排队并传递给 Claude
5. **会话持久化**：会话会持续到完成或被显式结束为止

## 仪表板交互

当 Claude 以 headless 模式运行时，你可以通过仪表板进行交互：

- **监控进度**：实时查看 Claude 的操作
- **回答问题**：当 Claude 需要输入时进行回复
- **查看更改**：查看文件修改的 git diff
- **控制会话**：结束会话或提供新的指令

## 日志

日志存储在：
```
~/.omnara/claude_headless/<session-id>.log
```

实时查看日志：
```bash
tail -f ~/.omnara/claude_headless/<session-id>.log
```
