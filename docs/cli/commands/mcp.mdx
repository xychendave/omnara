## 用法

```bash
omnara mcp [OPTIONS]
```

运行 MCP（Model Context Protocol）stdio 服务器，用于与 Claude Desktop、Cursor、Cline 等兼容 MCP 的客户端以及其他 AI 编程助手集成。

## 选项

<ParamField path="--api-key" type="string" required>
  用于身份验证的 API key
</ParamField>

<ParamField path="--permission-tool" type="boolean">
  启用 Claude Code 权限提示工具
</ParamField>

<ParamField path="--git-diff" type="boolean">
  为所有消息启用自动 git diff 捕获
</ParamField>

<ParamField path="--agent-instance-id" type="string">
  使用已有的智能体实例，而不是创建新实例
</ParamField>

<ParamField path="--disable-tools" type="boolean">
  禁用除权限工具外的所有工具
</ParamField>

## 示例

<CodeGroup>

```bash Basic
# 启动 MCP 服务器
omnara mcp --api-key YOUR_KEY
```

```bash Full Features
# 启用 git diff 跟踪和权限提示
omnara mcp --api-key YOUR_KEY --git-diff --permission-tool
```

```bash Permission Only
# 仅运行权限提示工具
omnara mcp --api-key YOUR_KEY --permission-tool --disable-tools
```

</CodeGroup>

## MCP 客户端配置

### Claude Desktop

编辑 `~/Library/Application Support/Claude/claude_desktop_config.json`（macOS）：

```json claude_desktop_config.json
{
  "mcpServers": {
    "omnara": {
      "command": "omnara",
      "args": ["mcp", "--api-key", "YOUR_API_KEY"]
    }
  }
}
```

### Cursor

编辑 `.cursorrules` 或 Cursor 设置：

```json
{
  "mcpServers": {
    "omnara": {
      "type": "stdio",
      "command": "omnara",
      "args": ["mcp", "--api-key", "YOUR_API_KEY", "--git-diff"]
    }
  }
}
```

### 使用 pipx

对于 pipx 安装方式：

```json
{
  "mcpServers": {
    "omnara": {
      "command": "pipx",
      "args": ["run", "--no-cache", "omnara", "mcp", "--api-key", "YOUR_API_KEY"]
    }
  }
}
```

## 可用工具

MCP 服务器向 AI 智能体提供以下工具：

<Tabs>
  <Tab title="log_step">
    **用途**：记录智能体正在执行的高层级步骤

    **参数：**
    - `step_description`（必填）：智能体正在做什么
    - `agent_instance_id`（可选）：实例 ID

    **返回值：**
    - `agent_instance_id`：会话标识符
    - `step_number`：递增的步骤计数
    - `user_feedback`：排队中的用户消息列表

    **关键特性**：每个步骤都会返回待处理的用户反馈
  </Tab>

  <Tab title="ask_question">
    **用途**：向用户提问并阻塞等待回答

    **参数：**
    - `agent_instance_id`（必填）：当前会话
    - `question_text`（必填）：要提出的问题

    **返回值：**
    - `answer`：用户的回复文本
    - `question_id`：消息 ID

    **行为**：轮询等待回复（默认 24 小时超时，10 秒间隔）
  </Tab>

  <Tab title="end_session">
    **用途**：将智能体会话标记为已完成

    **参数：**
    - `agent_instance_id`（必填）：要结束的会话

    **返回值：**
    - `success`：布尔值
    - `final_status`：应为 "completed"

    **效果**：将状态设置为 COMPLETED，并记录结束时间戳
  </Tab>

  <Tab title="approve（可选）">
    **用途**：Claude Code 权限提示处理器

    **启用方式**：`--permission-tool` 参数

    **参数：**
    - `tool_name`：请求权限的工具
    - `input`：工具参数

    **特性：**
    - 提供选项：是（仅此次）、是（本会话内）、否
    - 按实例在内存中缓存权限
    - 将提示格式化为 markdown

    **返回值**：`{"behavior": "allow"|"deny"}`
  </Tab>
</Tabs>

## Git Diff 捕获

启用 `--git-diff` 后：

<Steps>
  <Step title="初始化">
    在启动时捕获初始 git 提交哈希
  </Step>
  <Step title="生成 Diff">
    每次调用 `log_step` 和 `ask_question` 时，生成展示从初始提交到当前状态所有更改的 diff
  </Step>
  <Step title="排除 Worktree">
    自动将其他 git worktree 排除在 diff 之外
  </Step>
  <Step title="未跟踪文件">
    以类似 diff 的格式包含未跟踪的文件
  </Step>
</Steps>

**适用场景**：无需手动执行 git 命令即可自动跟踪代码更改。diff 会附加在发送到仪表板的每条消息上。

## 权限工具

通过 `--permission-tool` 启用，将 Claude Code 权限提示路由到 Omnara 仪表板：

```bash
omnara mcp --api-key YOUR_KEY --permission-tool
```

**工作流程：**
1. Claude Code 请求权限（例如"运行这个 bash 命令？"）
2. 请求通过 `approve` 工具发送到 Omnara MCP 服务器
3. 仪表板向用户显示权限提示
4. 用户通过仪表板批准/拒绝
5. 响应回传给 Claude Code
6. 权限决定在本会话内缓存

**优势：**
- 远程批准 Claude Code 的操作
- 在批准前审阅权限请求
- 会话级别的权限缓存

## 禁用工具

使用 `--disable-tools` 可禁用除权限工具外的所有工具：

```bash
omnara mcp --api-key YOUR_KEY --permission-tool --disable-tools
```

**适用场景**：当你仅将 Omnara 用于 Claude Code 的权限提示时，禁用日志/提问工具以降低开销。

## 工作原理

1. **Stdio 服务器**：使用 MCP 协议通过 stdin/stdout 通信
2. **工具注册**：向 MCP 客户端注册可用工具
3. **请求处理**：处理来自 AI 智能体的工具调用
4. **API 集成**：将消息转发到 Omnara API 进行存储/仪表板同步
5. **响应传递**：通过 stdio 将结果返回给 AI 智能体

## 身份验证

使用基于 JWT 的身份验证和自定义 RSA 密钥：

- API key 是带有 `user_id` 声明的 JWT 令牌
- 为缩短 API key 使用了较弱的 RSA 密钥（设计上的权衡）
- 所有操作均限定在已认证用户范围内
- 尽管加密强度较弱，密钥仍应妥善保管

<Warning>
请安全地存储 API key。虽然 MCP 服务器为缩短密钥长度使用了较弱的 RSA，但它们仍能提供对你 Omnara 账户的用户级访问权限。
</Warning>
