## 用法

```bash
omnara serve [OPTIONS]
```

启动一个 webhook 服务器，允许从 Omnara 仪表板或其他集成远程触发 Claude Code 会话。非常适合通过手机启动 AI 编程会话或自动化启动智能体。

## 选项

<ParamField path="--no-tunnel" type="boolean">
  在本地运行，不使用 Cloudflare 隧道（默认：启用隧道）
</ParamField>

<ParamField path="--port" type="number" default="6662">
  服务器端口
</ParamField>

<ParamField path="--permission-mode" type="string">
  传递给 Claude 实例的权限模式（选项与 `omnara headless` 相同）
</ParamField>

`omnara headless` 的其他权限相关参数同样受支持，并会透传给 Claude 实例。

## 示例

<CodeGroup>

```bash Default (with tunnel)
# 使用 Cloudflare 隧道启动（公开 URL）
omnara serve

# 输出会显示 webhook URL：
# ╔══════════════════════════════╗
# ║  Webhook URL: https://abc123.trycloudflare.com ║
# ║  API Key: xyz789...                            ║
# ╚══════════════════════════════╝
```

```bash Local Only
# 仅本地服务器，不使用隧道
omnara serve --no-tunnel --port 8080
```

```bash With Permissions
# 为所有实例设置默认权限模式
omnara serve --permission-mode acceptEdits
```

</CodeGroup>

## 设置指南

### 1. 启动服务器

```bash
omnara serve
```

服务器启动后会显示：
- **Webhook URL**：用于触发的公开 URL（如果启用了隧道）
- **API Key**：用于验证请求的密钥

### 2. 在仪表板中配置

1. 在你的[仪表板](https://claude.omnara.com/dashboard)中进入 **Dashboard → New Session**
2. 添加一个新的 webhook 集成
3. 粘贴 webhook URL
4. 保存 API key

### 3. 从仪表板触发

在仪表板中点击"Launch Agent"，即可在你的机器上触发一个新的 Claude Code 会话。

## 工作原理

<Steps>
  <Step title="webhook 服务器启动">
    在指定端口上监听 HTTP POST 请求
  </Step>
  <Step title="Cloudflare 隧道（可选）">
    如果未设置 `--no-tunnel`，则启动 Cloudflare 隧道以提供公开的 HTTPS URL
  </Step>
  <Step title="仪表板发送请求">
    当你从仪表板触发时，它会发送一个 POST 请求，包含：
    - 智能体实例 ID
    - 初始提示词
    - 分支/worktree 名称（可选）
  </Step>
  <Step title="创建 Git Worktree">
    服务器为该会话创建一个隔离的 git worktree（如果指定）
  </Step>
  <Step title="启动 Claude">
    在该 worktree 中启动一个 headless Claude Code 实例
  </Step>
  <Step title="后台执行">
    Claude 作为后台任务运行，通过仪表板进行通信
  </Step>
</Steps>

## Cloudflare 隧道

默认情况下，`omnara serve` 使用 Cloudflare 隧道暴露一个公开的 HTTPS URL，无需端口转发或防火墙配置。

### 使用隧道（默认）
```bash
omnara serve
# → 可通过 https://xxx.trycloudflare.com 从任何地方访问
```

**优势：**
- 无需端口转发
- HTTPS 加密
- 可在企业防火墙后工作
- 免费（使用 Cloudflare 的免费层）

### 不使用隧道（本地）
```bash
omnara serve --no-tunnel --port 8080
# → 仅可通过本地网络在 http://localhost:8080 访问
```

**适用情况：**
- 已配置反向代理的自托管环境
- 本地开发/测试
- 安全策略禁止使用外部隧道

<Tip>
Cloudflare 隧道需要安装 `cloudflared`。在 macOS 上，如果缺失会自动通过 Homebrew 安装。
</Tip>

## Git Worktree 隔离

webhook 服务器为每个会话创建隔离的 git worktree，允许多个 Claude 实例同时在不同分支上工作而互不冲突。

```bash
# 触发时创建 worktree：
git worktree add ./feature-auth -b feature-auth

# Claude 在隔离目录中工作：
./feature-auth/

# 主 worktree 保持干净：
./
```

**Worktree 模式：**
- **Main**：使用当前目录（不隔离）
- **Existing**：如果找到已有 worktree 则复用
- **New**：使用新分支创建全新的 worktree

## 安全性

### 身份验证
- webhook 密钥在启动时生成
- 包含在 webhook URL 中或单独发送
- 每个请求都会进行校验

### API Key 处理
- 每个被触发的实例都会获得唯一的 Omnara API key
- 密钥在存储前会进行哈希处理（SHA256）
- 仅限已认证用户范围内使用

### 网络安全
- Cloudflare 隧道提供 HTTPS 加密
- 本地模式需要手动配置 SSL
- webhook 密钥可防止未授权访问

## 故障排查

<AccordionGroup>
  <Accordion title="隧道启动失败">
    确保已安装 `cloudflared`：
    ```bash
    brew install cloudflared  # macOS
    ```

    或使用本地模式：
    ```bash
    omnara serve --no-tunnel
    ```
  </Accordion>

  <Accordion title="webhook 已触发但 Claude 未启动">
    检查日志：
    ```
    ~/.omnara/claude_headless/<session-id>.log
    ```

    确认已安装 Claude Code CLI：
    ```bash
    claude --version
    ```
  </Accordion>

  <Accordion title="Git worktree 错误">
    确保你位于一个 git 仓库中：
    ```bash
    git status
    ```

    或在请求中指定主 worktree 以跳过隔离
  </Accordion>
</AccordionGroup>
