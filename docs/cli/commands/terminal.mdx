## 用法

```bash
omnara terminal [OPTIONS] [AGENT_FLAGS]
```

启动一个启用了 WebSocket 中继流式传输的 AI 编码代理。

**代理标志透传**：所选代理（Claude Code、Codex、Amp）特有的所有标志都会自动传递给底层代理 CLI。这让你在保持 Omnara 集成的同时使用代理的原生功能。

## 选项

支持默认命令的所有选项：

<ParamField path="--agent" type="string" default="claude">
  选择要使用的代理：`claude`、`codex` 或 `amp`
</ParamField>

<ParamField path="--name" type="string">
  仪表板中显示的自定义名称
</ParamField>

<ParamField path="--api-key" type="string">
  用于身份验证的 API key
</ParamField>

<ParamField path="--agent-instance-id" type="string">
  恢复现有会话
</ParamField>

<ParamField path="--relay-url" type="string">
  自定义中继 WebSocket URL

  **默认值：**
  - 生产环境：`wss://relay.omnara.com/agent`
  - 本地开发：`ws://localhost:8080/agent`
</ParamField>

<ParamField path="--base-url" type="string" default="https://agent.omnara.com">
  Omnara API 服务器 URL
</ParamField>

<Note>
**代理标志兼容性**：所有代理特有的 CLI 标志都会透传给底层代理：
- **Claude Code**：`--continue`、`--permission-mode`、`--allowed-tools`、`--dangerously-skip-permissions` 等
- **Codex**：任何 Codex 特有的标志
- **Amp**：任何 Amp 特有的标志

这样既保留了代理的完整功能，又增加了 Omnara 的仪表板集成。
</Note>

## 示例

<CodeGroup>

```bash Basic
# 启动并启用中继流式传输（显式）
omnara terminal
```

```bash Named Session
# 以自定义名称启动 Codex
omnara terminal --agent codex --name "Feature Development"
```

```bash Custom Relay
# 使用自定义中继服务器
omnara terminal --relay-url wss://relay.mycompany.com/agent
```

```bash Resume Session
# 继续现有会话
omnara terminal --agent-instance-id abc-123-def
```

```bash Claude Code Flags
# 使用任意 Claude Code 标志
omnara terminal --permission-mode plan --allowed-tools Read,Write,Bash

# 继续之前的 Claude 会话
omnara terminal --continue

# 组合使用 Omnara 和 Claude 的标志
omnara terminal --name "Code Review" --permission-mode acceptEdits
```

```bash Codex Flags
# 使用 Codex 及其原生标志
omnara terminal --agent codex [CODEX_FLAGS]
```

```bash Amp Flags
# 使用 Amp 及其原生标志
omnara terminal --agent amp [AMP_FLAGS]
```

</CodeGroup>

<Tip>
**代理标志示例：**
- **Claude Code**：`--continue`、`--permission-mode`、`--allowed-tools`、`--disallowed-tools`、`--dangerously-skip-permissions`
- **Codex**：任何 Codex 特有的配置标志
- **Amp**：任何 Amp 特有的配置标志

所有代理标志都会自动透传，以保持完整的原生功能。
</Tip>

## 什么是 WebSocket 中继？

中继系统将你的终端 I/O 实时流式传输到 Omnara 仪表板，从而实现：

- **实时终端查看**：从网页/移动端准确查看终端中正在发生的一切
- **远程输入**：在仪表板上输入消息并显示在终端中
- **会话共享**：与队友共享终端会话
- **多设备访问**：用手机监控，用笔记本电脑控制

## 工作原理

<Steps>
  <Step title="代理注册">
    CLI 向 Omnara API 注册代理实例
  </Step>

  <Step title="WebSocket 连接">
    使用 API key 身份验证连接到配置 URL 上的中继服务器
  </Step>

  <Step title="PTY 创建">
    创建伪终端（PTY）并启动代理
  </Step>

  <Step title="双向流式传输">
    通过 WebSocket 流式传输终端 I/O：
    - **终端 → 仪表板**：实时显示输出
    - **仪表板 → 终端**：来自网页/移动端的用户输入
  </Step>

  <Step title="会话持久化">
    终端历史记录会被缓冲，并在从新设备查看时重放
  </Step>
</Steps>

## 中继模式 vs 仅本地模式

<Tabs>
  <Tab title="使用中继（terminal）">
    ```bash
    omnara terminal
    # 或者直接使用：omnara
    ```

    **功能：**
    - ✓ 实时终端流式传输
    - ✓ 仪表板显示实时终端输出
    - ✓ 支持来自网页/移动端的输入
    - ✓ 会话共享
    - ✓ 多设备监控
    - ✓ 消息 API 处于激活状态
  </Tab>

  <Tab title="不使用中继（--no-relay）">
    ```bash
    omnara --no-relay
    ```

    **功能：**
    - ✗ 无终端流式传输
    - ✗ 仪表板仅显示消息
    - ✗ 不支持来自网页/移动端的输入
    - ✗ 无会话共享
    - ✗ 仅可通过终端访问
    - ✓ 消息 API 仍处于激活状态

    **适用场景：**
    - 敏感代码库（不进行终端流式传输）
    - 离线开发
    - 减少带宽占用
  </Tab>
</Tabs>

## 终端帧协议

中继使用二进制帧协议实现高效的流式传输：

| 帧类型 | ID | 用途 |
|------------|-----|---------|
| OUTPUT | 0 | 发送到仪表板的终端输出 |
| INPUT | 1 | 来自仪表板的用户输入 |
| RESIZE | 2 | 窗口大小调整事件 |
| METADATA | 3 | 代理元数据（类型、名称） |

**帧结构：**
```
[Type: 1 byte][Length: 4 bytes][Payload: variable]
```

该协议确保了高效、低延迟的终端流式传输。

## 特殊功能

### 历史记录重放

当你打开仪表板时：
- 重放缓冲的终端历史记录
- 对 Codex 进行特殊处理（去除清屏序列）
- 无缝同步到当前状态

### 窗口大小调整处理

终端大小调整会被智能处理：
- **防抖动**：等待 500ms 后再应用大小调整
- **渐进式调整**：对于较大的差异分步进行变更
- **光标跟踪**：检测并调整光标位置报告

防止 TUI 应用出现显示错乱。

### 重新连接

如果 WebSocket 连接断开：
- 每 5 秒自动重新连接
- 会话在重连期间保持不变
- 断开连接期间不会丢失数据

## 中继服务器架构

中继服务器（`relay.omnara.com`）：
- 独立于 API 服务器运行
- 处理 WebSocket 连接
- 缓冲终端历史记录
- 在终端和仪表板客户端之间路由消息
- 支持多个查看者同时在线

## 安全性

### 身份验证

WebSocket 连接通过以下方式进行身份验证：
- `X-API-Key` 请求头
- `Authorization: Bearer {token}` 请求头
- WebSocket 子协议：`omnara-key.{key}` 或 `omnara-supabase.{token}`

### 加密

- 生产环境：基于 TLS 的 WSS（WebSocket Secure）
- 所有终端数据在传输过程中均加密
- 不长期存储终端数据（仅在内存中缓冲）

### 访问控制

- 只有经过身份验证的用户才能查看自己的会话
- 加入会话时验证会话所有权
- 不允许跨用户访问会话

## 故障排查

<AccordionGroup>
  <Accordion title="中继连接失败">
    **检查：**
    - 网络/防火墙允许 WebSocket 连接
    - API key 有效：`omnara --auth`
    - 中继 URL 正确

    **手动测试：**
    ```bash
    # 尝试使用自定义中继 URL
    omnara terminal --relay-url wss://relay.omnara.com/agent
    ```
  </Accordion>

  <Accordion title="终端未在仪表板中显示">
    **原因：**
    - 使用了 `--no-relay` 标志
    - WebSocket 连接已断开
    - 仪表板未刷新

    **解决方法：**
    1. 确保未使用 `--no-relay`
    2. 检查日志：`~/.omnara/<agent>_wrapper/<session-id>.log`
    3. 刷新仪表板
  </Accordion>

  <Accordion title="仪表板的输入不起作用">
    **原因：**
    - 连接仅为单向
    - 代理已退出或崩溃
    - 中继服务器故障

    **解决方法：**
    - 检查代理是否仍在终端中运行
    - 尝试重新启动会话
    - 检查中继服务器状态
  </Accordion>
</AccordionGroup>
