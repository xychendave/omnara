## 配置目录

所有 Omnara 配置文件都存储在：

```
~/.omnara/
```

该目录在首次运行时自动创建，并使用安全权限（`0o700`——仅所有者可访问）。

## credentials.json

**位置：** `~/.omnara/credentials.json`

**用途：** 存储身份验证凭据（API key）

**格式：**
```json
{
  "write_key": "xxxxxxxxxxxxxxxxxxxx"
}
```

**权限：** `0o600`（仅所有者可读写）

### 创建方式

当你进行身份验证时，API key 会被自动保存：

```bash
omnara --auth
```

浏览器身份验证流程会将密钥保存到此文件。

### 手动管理

你也可以手动设置 API key：

```bash
# 使用 jq
echo '{"write_key": "YOUR_KEY_HERE"}' | jq > ~/.omnara/credentials.json

# 或直接编辑
nano ~/.omnara/credentials.json
```

<Warning>
请妥善保管此文件！它包含你的身份验证凭据。切勿将其提交到版本控制系统。
</Warning>

## config.json

**位置：** `~/.omnara/config.json`

**用途：** 存储用户偏好（非敏感设置）

**格式：**
```json
{
  "default_agent": "claude"
}
```

**权限：** `0o600`（与凭据文件保持一致）

### 创建方式

当你设置偏好时，配置会被保存：

```bash
# 设置默认智能体
omnara --set-default codex
```

这会创建/更新 `config.json` 中的偏好设置。

### 可用设置

当前支持的设置：

<ParamField path="default_agent" type="string">
  要使用的默认智能体：`claude`、`codex` 或 `amp`

  ```json
  {
    "default_agent": "codex"
  }
  ```
</ParamField>

未来版本可能会添加更多设置。

### 手动编辑

你可以直接编辑 config.json：

```bash
nano ~/.omnara/config.json
```

CLI 会将新设置与现有设置合并，保留其他值。

## 日志文件

智能体会话会在 `~/.omnara/` 的子目录中创建日志文件：

### 位置模式

```
~/.omnara/<agent-type>/<session-id>.log
```

### 示例

<CodeGroup>

```plaintext Claude Code
~/.omnara/claude_wrapper/abc-123-def-456.log
```

```plaintext Codex
~/.omnara/codex_wrapper/abc-123-def-456.log
```

```plaintext Amp
~/.omnara/amp_wrapper/abc-123-def-456.log
```

```plaintext Headless
~/.omnara/claude_headless/abc-123-def-456.log
```

</CodeGroup>

### 查看日志

```bash
# 从仪表盘或 CLI 输出中找到你的会话 ID
export SESSION_ID="abc-123-def-456"

# 跟踪日志文件
tail -f ~/.omnara/claude_wrapper/$SESSION_ID.log

# 或搜索最新的日志
ls -t ~/.omnara/claude_wrapper/*.log | head -1 | xargs tail -f
```

## 文件分离

<Tabs>
  <Tab title="credentials.json">
    **包含：**
    - API key
    - 身份验证令牌
    - 敏感机密信息

    **为何分离：**
    - 可以从备份中排除
    - 不同的访问控制
    - 永不共享或纳入版本控制
  </Tab>

  <Tab title="config.json">
    **包含：**
    - 用户偏好
    - 默认设置
    - 非敏感配置

    **为何分离：**
    - 可以在多台机器之间同步
    - 可以安全备份
    - 可以纳入版本控制（如需要）
  </Tab>
</Tabs>

## 备份与迁移

### 备份

```bash
# 备份配置目录
cp -r ~/.omnara ~/.omnara.backup

# 或仅备份凭据
cp ~/.omnara/credentials.json ~/.omnara/credentials.json.backup
```

### 迁移到新机器

```bash
# 在旧机器上
tar -czf omnara-config.tar.gz ~/.omnara/credentials.json ~/.omnara/config.json

# 将文件传输到新机器

# 在新机器上
tar -xzf omnara-config.tar.gz -C ~/
```

<Tip>
只需迁移 `credentials.json` 和 `config.json`。日志文件与具体会话相关，无需传输。
</Tip>

## 故障排除

### credentials.json 损坏

如果你看到身份验证错误：

```bash
# 检查文件内容
cat ~/.omnara/credentials.json

# 验证 JSON 语法
python -m json.tool ~/.omnara/credentials.json

# 如果损坏，重新进行身份验证
omnara --reauth
```

### 权限错误

如果你看到权限错误：

```bash
# 修复目录权限
chmod 700 ~/.omnara

# 修复文件权限
chmod 600 ~/.omnara/credentials.json
chmod 600 ~/.omnara/config.json
```

### 重置配置

如需从头开始：

```bash
# 先备份（可选）
mv ~/.omnara ~/.omnara.backup

# 删除配置
rm -rf ~/.omnara

# 重新进行身份验证
omnara --auth
```

## 安全最佳实践

<AccordionGroup>
  <Accordion title="切勿提交凭据" icon="git">
    添加到你的 `.gitignore`：
    ```
    .omnara/
    **/credentials.json
    ```
  </Accordion>

  <Accordion title="在 CI/CD 中使用环境变量" icon="server">
    不要在 CI/CD 系统中存储 `credentials.json`。请使用 secrets：
    ```bash
    export OMNARA_API_KEY="${{ secrets.OMNARA_API_KEY }}"
    ```
  </Accordion>

  <Accordion title="安全备份" icon="shield">
    如果需要备份凭据：
    ```bash
    # 备份前先加密
    tar -czf - ~/.omnara | gpg -c > omnara-backup.tar.gz.gpg
    ```
  </Accordion>

  <Accordion title="定期轮换密钥" icon="rotate">
    定期从仪表盘生成新的 API key 并更新凭据：
    ```bash
    omnara --reauth
    ```
  </Accordion>
</AccordionGroup>
