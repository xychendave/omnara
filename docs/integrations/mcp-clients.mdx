## 概述

Omnara 提供一个 MCP（Model Context Protocol）服务器，可与多种 AI 编程助手集成，包括 Claude Desktop、Cursor、Cline、Windsurf 等。

## 快速设置

### Claude Desktop

编辑 `~/Library/Application Support/Claude/claude_desktop_config.json`（macOS）或 `%APPDATA%\Claude\claude_desktop_config.json`（Windows）：

```json
{
  "mcpServers": {
    "omnara": {
      "command": "omnara",
      "args": ["mcp", "--api-key", "YOUR_API_KEY"]
    }
  }
}
```

### Cursor

添加到你的 Cursor 设置或 `.cursorrules` 中：

```json
{
  "mcpServers": {
    "omnara": {
      "type": "stdio",
      "command": "omnara",
      "args": ["mcp", "--api-key", "YOUR_API_KEY", "--git-diff"]
    }
  }
}
```

### 使用 pipx

如果你是通过 pipx 安装的 Omnara：

```json
{
  "mcpServers": {
    "omnara": {
      "command": "pipx",
      "args": ["run", "--no-cache", "omnara", "mcp", "--api-key", "YOUR_API_KEY"]
    }
  }
}
```

## 可用工具

将 Omnara 添加为 MCP 服务器后，AI 智能体可以使用以下工具：

<Tabs>
  <Tab title="log_step">
    **用途**：记录进度更新

    **使用场景**：跟踪智能体正在做什么
    - 「正在分析代码库结构……」
    - 「找到 5 个需要审查的文件」
    - 「正在修改 authentication.py」
  </Tab>

  <Tab title="ask_question">
    **用途**：向用户提问

    **使用场景**：在需要时获取人工输入
    - 「我可以继续进行这次重构吗？」
    - 「你更倾向于哪种方案？」
    - 「我可以删除这个已弃用的文件吗？」
  </Tab>

  <Tab title="end_session">
    **用途**：标记工作完成

    **使用场景**：清晰的会话跟踪
    - 将会话标记为 COMPLETED
    - 显示在仪表板历史记录中
  </Tab>

  <Tab title="approve（可选）">
    **用途**：权限提示

    **使用场景**：为 Claude Code 提供远程批准
    - 通过 `--permission-tool` 启用
    - 将权限提示路由到仪表板
    - 随时随地批准操作
  </Tab>
</Tabs>

## 高级配置

### 启用 Git Diff 跟踪

```json
{
  "mcpServers": {
    "omnara": {
      "command": "omnara",
      "args": ["mcp", "--api-key", "YOUR_API_KEY", "--git-diff"]
    }
  }
}
```

自动在每条消息中捕获代码变更。

### 权限工具模式

```json
{
  "mcpServers": {
    "omnara": {
      "command": "omnara",
      "args": [
        "mcp",
        "--api-key", "YOUR_API_KEY",
        "--permission-tool",
        "--disable-tools"
      ]
    }
  }
}
```

仅将 Omnara 用于权限提示，禁用其他工具以减少开销。

### 恢复已有会话

```json
{
  "mcpServers": {
    "omnara": {
      "command": "omnara",
      "args": [
        "mcp",
        "--api-key", "YOUR_API_KEY",
        "--agent-instance-id", "abc-123-def-456"
      ]
    }
  }
}
```

继续一个已有会话，而不是创建新会话。

## 工作原理

<Steps>
  <Step title="客户端启动 MCP 服务器">
    你的 AI 客户端（Claude Desktop、Cursor 等）以子进程方式启动 `omnara mcp`
  </Step>

  <Step title="注册工具">
    Omnara 通过 stdio 协议向 MCP 客户端注册可用工具
  </Step>

  <Step title="智能体使用工具">
    AI 智能体按需调用工具：
    - `log_step("Analyzing code...")`
    - `ask_question("Proceed with changes?")`
  </Step>

  <Step title="同步到仪表板">
    所有消息和提问会实时显示在你的 Omnara 仪表板上
  </Step>

  <Step title="用户回复">
    你可以通过以下方式回复提问：
    - Web 仪表板
    - 移动应用
    - 任何可访问 Omnara 的设备
  </Step>

  <Step title="回复送达">
    你的回复会通过 MCP 服务器发送回 AI 智能体
  </Step>
</Steps>

## 使用场景

<AccordionGroup>
  <Accordion title="移动端监督" icon="mobile">
    通过手机监控 Claude Desktop 正在做什么。不在电脑旁时也能回复提问。

  </Accordion>
</AccordionGroup>

## 故障排除

<AccordionGroup>
  <Accordion title="MCP 服务器无法连接">
    **检查：**
    1. Omnara 已安装：`omnara --version`
    2. API key 有效：`omnara mcp --api-key YOUR_KEY`（手动测试）
    3. 配置文件语法正确（合法的 JSON）
    4. 命令或参数没有拼写错误

    **调试：**
    ```bash
    # 手动测试 MCP 服务器
    omnara mcp --api-key YOUR_KEY

    # 查看日志
    # Claude Desktop: ~/Library/Logs/Claude/mcp.log
    ```
  </Accordion>

  <Accordion title="工具未显示">
    **原因：**
    - 客户端不支持 MCP 工具
    - 配置未重新加载
    - 客户端使用了错误的传输类型

    **修复：**
    1. 重启你的 AI 客户端
    2. 确认客户端支持 MCP stdio
    3. 在客户端日志中验证工具注册情况
  </Accordion>

  <Accordion title="消息未出现在仪表板">
    **原因：**
    - API key 错误
    - 网络/防火墙阻拦
    - base URL 错误（自托管场景）

    **修复：**
    ```bash
    # 手动测试 API key
    curl -H "Authorization: Bearer YOUR_KEY" \
      https://agent.omnara.com/api/v1/auth/verify
    ```
  </Accordion>
</AccordionGroup>
