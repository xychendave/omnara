## 概述

Omnara 的 n8n 集成（`n8n-nodes-omnara`）使工作流能够通过 Omnara 平台与用户进行实时沟通。非常适合审批工作流、智能体对话以及引导式自动化。

**核心能力：**
- 发送非阻塞的状态更新
- 提问并等待回复
- 兼容 AI Agent
- 多渠道通知（邮件、短信、推送）
- 会话管理

## 安装

在你的 n8n 实例中安装该社区节点：

```bash
npm install n8n-nodes-omnara
```

或通过 n8n 界面安装：
1. 进入 **Settings** → **Community Nodes**
2. 搜索 `n8n-nodes-omnara`
3. 点击 **Install**

<Card title="在 npm 上查看" icon="npm" href="https://www.npmjs.com/package/n8n-nodes-omnara">
  完整的包文档和更新日志
</Card>

## 快速开始

<Steps>
  <Step title="添加凭据">
    1. 在 n8n 中进入 **Credentials** → **New**
    2. 选择 **Omnara API**
    3. 输入从 [claude.omnara.com/dashboard](https://claude.omnara.com/dashboard) 获取的 API key
    4. 如使用自托管实例，可选填自定义服务器 URL
  </Step>

  <Step title="添加 Omnara 节点">
    将 **Omnara** 节点拖入你的工作流
  </Step>

  <Step title="配置操作">
    选择一个操作：
    - **Send Message**：非阻塞的状态更新
    - **Send and Wait**：提问并暂停工作流
    - **End Session**：将智能体会话标记为完成
  </Step>

  <Step title="进行测试">
    执行你的工作流，并在 [Omnara 仪表板](https://claude.omnara.com/dashboard)中查看消息
  </Step>
</Steps>

## 操作

### Send Message（非阻塞）

发送信息性消息，无需等待回复。

**适用场景：**
- 进度更新
- 状态通知
- 记录智能体操作

**示例：**
```javascript
{
  "operation": "send",
  "agentType": "Deployment Agent",
  "message": "Build completed successfully ✅",
  "options": {
    "sendPush": true
  }
}
```

**返回值：**
- `message_id`：所创建消息的 ID
- `agent_instance_id`：会话标识符
- `queued_user_messages`：所有待处理的用户回复

<Tip>
每次发送时都检查 `queued_user_messages`，以捕获用户在两次操作之间发送的回复。
</Tip>

### Send and Wait（阻塞）

提出问题并暂停工作流，直到用户回复。

**适用场景：**
- 审批工作流
- 需要用户输入
- 决策节点

**两种模式：**

<Tabs>
  <Tab title="Webhook 模式（默认）">
    **适用于常规工作流**

    - 使用 n8n 的 webhook 系统
    - 高效、事件驱动
    - 用户回复后恢复工作流
    - 最长等待时间：7 天

    ```javascript
    {
      "operation": "sendAndWait",
      "message": "Approve deployment to production?",
      "limitWaitTime": true,
      "limitType": "afterTimeInterval",
      "resumeAmount": 1,
      "resumeUnit": "hours"
    }
    ```
  </Tab>

  <Tab title="同步模式（AI Agent）">
    **适用于 AI Agent 工具**

    - 同步轮询
    - 可在 AI Agent 上下文中工作
    - n8n AI Agent 必须使用此模式
    - 最长等待时间：48 小时

    ```javascript
    {
      "operation": "sendAndWait",
      "message": "Should I proceed with this change?",
      "options": {
        "syncMode": true,
        "syncTimeout": 600,  // 10 分钟
        "pollInterval": 5     // 每 5 秒检查一次
      }
    }
    ```

    <Warning>
    **AI Agent 必须使用 `syncMode: true`**。常规 webhook 模式在 AI Agent 上下文中无法工作。
    </Warning>
  </Tab>
</Tabs>

### End Session

将智能体会话标记为已完成。

```javascript
{
  "operation": "endSession",
  "agentInstanceId": "{{ $json.agent_instance_id }}"
}
```

## 配置示例

### 简单的审批工作流

```javascript
{
  "nodes": [
    {
      "name": "Build App",
      "type": "n8n-nodes-base.executeCommand",
      "parameters": {
        "command": "npm run build"
      }
    },
    {
      "name": "Request Approval",
      "type": "n8n-nodes-omnara.omnara",
      "parameters": {
        "operation": "sendAndWait",
        "agentType": "Deployment Bot",
        "message": "Build complete. Deploy to production?",
        "options": {
          "sendEmail": true,
          "sendPush": true
        },
        "limitWaitTime": true,
        "resumeAmount": 1,
        "resumeUnit": "hours"
      }
    },
    {
      "name": "Deploy",
      "type": "n8n-nodes-base.executeCommand",
      "parameters": {
        "command": "npm run deploy"
      }
    }
  ]
}
```

### AI Agent 集成

```javascript
{
  "nodes": [
    {
      "name": "AI Agent",
      "type": "@n8n/n8n-nodes-langchain.agent",
      "parameters": {
        "tools": ["omnara"],
        "prompt": "You can ask the user questions using the omnara tool."
      }
    },
    {
      "name": "Omnara Tool",
      "type": "n8n-nodes-omnara.omnara",
      "parameters": {
        "operation": "sendAndWait",
        "message": "{{ $json.question }}",
        "options": {
          "syncMode": true,        // AI Agent 必需
          "syncTimeout": 600,
          "pollInterval": 5,
          "sendEmail": true
        }
      }
    }
  ]
}
```

## 智能体实例管理

### 会话跟踪

每次工作流执行应使用一致的 `agent_instance_id`：

**模式 1：Webhook 触发（推荐）**
```javascript
// Omnara 仪表板使用预生成的 ID 触发工作流
{
  "agent_instance_id": "{{ $json.agent_instance_id }}", // 来自 webhook
  "agent_type": "{{ $json.agent_type }}"
}
```

**模式 2：自行生成**
```javascript
// 在 Set 节点中生成 UUID
{
  "agent_instance_id": "{{ $uuid() }}",
  "agent_type": "My Workflow"
}
// 在所有后续 Omnara 节点中引用
```

### 排队消息

Omnara 在每条智能体消息中都会返回排队的用户消息：

```javascript
const response = await omnaraNode.execute();
// response.queued_user_messages 包含自上次检查以来的所有用户回复
```

这确保你不会错过任何用户输入，即使时机不凑巧也没关系。

控制如何通知用户：

</ParamField>

  发送推送通知（通过移动应用或网页推送）

**优先级：**
1. 消息级覆盖（在请求中指定）
2. 用户偏好设置（在仪表板中配置）
3. 默认：不发送通知

## 错误处理

```javascript
{
  "nodes": [
    {
      "name": "Omnara",
      "type": "n8n-nodes-omnara.omnara",
      "continueOnFail": true,
      "parameters": {
        "operation": "send",
        "message": "Status update"
      }
    }
  ]
}
```

设置 `continueOnFail: true` 后，错误会以数据形式返回：

```javascript
{
  "error": "Failed to send message: Connection timeout",
  "resource": "message",
  "operation": "send",
  "itemIndex": 0
}
```

## 故障排除

<AccordionGroup>
  <Accordion title="Webhook 未恢复工作流">
    **原因：**
    - 在常规工作流中使用了同步模式（应使用 webhook 模式）
    - 消息元数据中的 webhook URL 不正确
    - 工作流执行已经超时

    **修复：**
    ```javascript
    // 在常规工作流中确保 sync 模式为 false（或不设置）
    {
      "options": {
        "syncMode": false  // 或完全省略
      }
    }
    ```
  </Accordion>

  <Accordion title="AI Agent 工具不起作用">
    **原因：**
    - 缺少 `syncMode: true`
    - 在 AI Agent 上下文中使用了 webhook 模式

    **修复：**
    ```javascript
    // AI Agent 必须使用同步模式
    {
      "options": {
        "syncMode": true,
        "syncTimeout": 600,
        "pollInterval": 5
      }
    }
    ```
  </Accordion>

  <Accordion title="消息未出现在仪表板中">
    **原因：**
    - API key 不正确
    - 服务器 URL 错误
    - 网络/防火墙问题

    **修复：**
    1. 测试凭据：确认凭据测试通过
    2. 验证服务器 URL（默认值：`https://agent.omnara.com`）
    3. 查看 n8n 日志获取错误详情
  </Accordion>

  <Accordion title="等待回复超时">
    **原因：**
    - 用户未在规定时间内回复
    - 轮询间隔过长
    - 同步超时时间过短

    **修复：**
    ```javascript
    // 调整超时设置
    {
      "limitWaitTime": true,
      "resumeAmount": 2,       // 改为 2 小时而不是 1 小时
      "resumeUnit": "hours",
      "options": {
        "syncTimeout": 1200,   // 20 分钟
        "pollInterval": 3      // 每 3 秒检查一次
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## API 端点

n8n 节点使用以下 Omnara API 端点：

| 端点 | 方法 | 用途 |
|----------|--------|---------|
| `/api/v1/messages/agent` | POST | 发送消息（send 与 sendAndWait 共用） |
| `/api/v1/messages/pending` | GET | 轮询用户回复（同步模式） |
| `/api/v1/sessions/end` | POST | 结束智能体会话 |
| `/api/v1/auth/verify` | GET | 验证凭据 |

<AccordionGroup>
  <Accordion title="使用具有描述性的智能体类型" icon="tag">
    ```javascript
    // 好的写法
    agentType: "Deployment Bot - Production"

    // 不好的写法
    agentType: "Bot"
    ```

    在运行多个工作流时，有助于在仪表板中识别会话。
  </Accordion>

  <Accordion title="设置合适的超时时间" icon="clock">
    ```javascript
    // 用于紧急审批
    resumeAmount: 30
    resumeUnit: "minutes"

    // 用于非紧急请求
    resumeAmount: 24
    resumeUnit: "hours"
    ```

    在用户便利性和工作流效率之间取得平衡。
  </Accordion>

  <Accordion title="始终结束会话" icon="check">
    ```javascript
    // 添加到工作流末尾
    {
      "name": "End Session",
      "operation": "endSession",
      "agentInstanceId": "{{ $('Request Approval').item.json.agent_instance_id }}"
    }
    ```

    保持仪表板整洁，并将会话标记为完成。
  </Accordion>

  <Accordion title="处理排队消息" icon="inbox">
    ```javascript
    // 每次发送时都检查用户回复
    const response = await omnaraNode.send(message);
    if (response.queued_user_messages.length > 0) {
      // 处理用户反馈
      const userMessage = response.queued_user_messages[0];
      console.log('User said:', userMessage.content);
    }
    ```

    确保不会错过用户在两次操作之间发送的输入。
  </Accordion>
</AccordionGroup>
